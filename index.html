Watch your repo come alive

Six features branch off the main line — everything you touch day to day, rendered visually instead of buried in flags.

Live repository tree

Branches and collaborators render as an animated, dynamic tree — with ahead/behind counts and diff stats on every card, at a glance.

Real-time push notifications

GitHub webhooks paired with a smee.io relay — falling back to polling automatically — deliver native desktop alerts the moment a watched branch updates.

Integrated Git operations

Clone, pull, fetch, commit, push and stash from one sync bar — with edge-case handling for detached HEAD, divergent branches and dirty worktrees.

Monaco-powered merge editor

Resolve three-way conflicts hunk by hunk with Accept Current, Accept Incoming or Accept Both — right inside the app, no terminal required.

commit f4c9e8b

Enterprise-grade security, locally.

Code Companion runs completely locally. Your code and credentials never leave your machine without your permission. We built in multiple layers of defense to keep your secrets safe.

1. OS-Level Native Encryption

We use Electron's built-in safeStorage API. Before your key is ever written to your hard drive, it gets encrypted using your operating system's native keychain (such as Keychain Access on macOS, Credential Manager on Windows, or libsecret / KWallet on Linux). It is never saved in plain text.

2. Device-Specific Fallback Encryption

If your Linux desktop environment doesn't have a keychain running, the app has a built-in fallback. It encrypts your key using military-grade encryption (AES-256-GCM) with a unique key derived from your computer's hostname and username. Even in fallback mode, the file cannot be decrypted if copied to another machine.

3. Pre-Commit Secret Scanning

To prevent accidental leaks, the app has an active safeguard built-in. When you try to commit your changes, the app scans your files for any accidental inclusions of passwords, API keys, or tokens. If it detects a potential secret, it blocks the commit and warns you in the UI so you can remove it before pushing to GitHub.
