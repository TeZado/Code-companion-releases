commit f4c9e8b

Enterprise-grade security, locally.

Code Companion runs completely locally. Your code and credentials never leave your machine without your permission. We built in multiple layers of defense to keep your secrets safe.

1. Native credential vault

Before a key ever touches your disk, it passes through your operating system's own secure credential store (such as Keychain Access on macOS, Credential Manager on Windows, or libsecret / KWallet on Linux). It is never saved in plain text.

2. Device-locked fallback encryption

If your desktop environment doesn't have a credential vault running, a built-in fallback takes over. It encrypts your key with military-grade encryption, using a unique key derived from your own machine. Copy the file to another computer and it simply won't open.

3. Pre-commit secret scanning

Every commit is scanned first. If a password, API key, or token slips into your changes, the commit is blocked and flagged in the UI before it ever reaches GitHub.
